Una empresa donde la junta directiva delega y un equipo de agentes ejecuta.

Estamos construyendo una plataforma multi-agente con UI propia que se monta encima de Hermes Agent. La idea: que cualquier persona de tu junta directiva pueda darle una instrucción a un agente CEO en lenguaje natural, y que ese CEO descomponga el trabajo y lo asigne a un equipo de agentes especializados que tú mismo configuras.

01 · Visión del producto

¿En qué se diferencia de Paperclip?

Paperclip popularizó la idea de "contratar" agentes. Nosotros tomamos esa misma promesa pero le agregamos tres cosas: una capa de organización (como una empresa real con junta directiva), un motor abierto y extensible (Hermes), y una UI que se siente menos a "consola técnica" y más a producto editorial.

02 · Cómo funciona

Una persona pide. El CEO descompone. Los workers ejecutan.

Este es el flujo en su versión más simple. Los detalles técnicos (Postgres, SQLite de Hermes, gateway en :8642) están abajo en la sección de arquitectura, pero para validar el producto solo importa este loop.

El humano habla. El CEO orquesta. Los workers ejecutan. Cuando hay duda, el sistema vuelve a preguntar.

03 · El recorrido de una tarea

Cinco estaciones desde "se me ocurrió" hasta "está hecho"

Cualquier tarea pasa por estas etapas. Las puedes ver en el tablero como columnas.

1

Triage

El CEO captura la idea pero todavía no la ha descompuesto. Aquí caen tareas nuevas.

2

To Do

Tarea descompuesta y lista para asignar. Tú o el CEO eligen al agente adecuado.

3

Ready

Asignada a un worker. El dispatcher de Hermes la moverá a Running cuando haya capacidad.

4

Running / Blocked

Worker trabajando. Si encuentra una decisión crítica, la bloquea esperando tu aprobación.

5

Done

Tarea completada con resumen. Notificación a la junta directiva.

04 · Roles y permisos

Tres tipos de usuarios. Tres niveles de poder.

Esta sección es crítica de validar. Si esperabas más roles (admin, finance, viewer, guest…) o un sistema de permisos granular, avísanos antes de que codifiquemos.

Owner

El que crea la organización. Control total, incluida la posibilidad de eliminarla.

  • ✓ Crear y eliminar la organización
  • ✓ Todas las acciones de Junta
  • ✓ Configurar facturación (futuro)

Junta Directiva

Quienes delegan al CEO y aprueban decisiones críticas. El día a día del producto.

  • ✓ Chatear con el CEO
  • ✓ Crear / aprobar tareas
  • ✓ Crear y editar agentes
  • ✓ Configurar MCP y webhooks
  • ✕ No puede borrar la organización

Member

Visibilidad limitada. Pensado para colaboradores futuros que solo necesiten ver progreso.

  • ✓ Ver proyectos asignados
  • ✕ No puede crear ni aprobar
  • ✕ No accede al chat del CEO
  • — rol futuro · v2 —

05 · Las páginas que vas a usar

Cuatro pantallas. El 90% del trabajo pasa por ellas.

Mockups simplificados. El detalle visual final ya está implementado en el design system Inteliside (Fraunces · Geist · teal sobre warm ivory/ink).

Pantalla principal

Dashboard / Kanban + Chat

A la izquierda el sidebar (organizaciones, navegación). En el centro el board. A la derecha el chat con el CEO. Todo a la vista, todo al alcance.

Pantalla de equipo

Agentes

Grid de cards con cada agente. Cada card muestra rol (CEO / C-Level / Worker), skills, MCP servers conectados y un toggle activo/inactivo.

Pantalla de configuración

Editor de Agente

Dos columnas: básico (nombre, rol, SOUL.md, skills, tools) y avanzado (MCP servers, webhooks, integraciones API).

Onboarding

Crear organización

Tres pasos rápidos: nombre + slug, objetivo principal, primer proyecto. Al final el CEO Agent se crea automáticamente y queda listo.

06 · Roadmap

Cuatro fases. ~13 semanas hasta producción.

Las primeras dos fases son la columna vertebral; las dos últimas suman superpoderes y pulido.

Fase 1 — Base

4–6 semanas en progreso
  • Setup Next.js · Better Auth · Postgres · Inteliside DS
  • CRUD organizaciones, proyectos, miembros
  • Integración Hermes Gateway API + CEO Profile
  • Kanban visual con drag & drop + sincronización Hermes ↔ Postgres
  • Chat con CEO + tool calling

Fase 2 — Equipo de agentes

3–4 semanas siguiente
  • CRUD agentes con templates predefinidos (CEO, C-Level, Workers)
  • UI para configurar MCP servers y skills
  • Creación dinámica de Hermes profiles
  • Workers funcionales con tools y skills

Fase 3 — Human-in-the-loop

2–3 semanas después
  • Sistema de aprobaciones para tareas bloqueadas
  • Notificaciones in-app y por email
  • Webhooks salientes
  • Integraciones Telegram / Slack

Fase 4 — Pulido y producción

2 semanas final
  • Optimización SSE / WebSocket
  • Manejo robusto de errores
  • Suite E2E completa
  • Documentación + onboarding pulido + deploy

09 · Riesgos

Lo que nos puede salir mal — y cómo lo evitamos

Impacto alto

Inestabilidad de Hermes Gateway

Probabilidad media. Mitigación: fallback a SQLite directo + retry con backoff exponencial.

Impacto medio

Sincronización fuera de fase

Probabilidad media. Mitigación: health checks + reconciliación periódica + audit log.

Impacto alto

Sobrecarga de agentes simultáneos

Probabilidad baja. Mitigación: rate limiting + queue del dispatcher.

Impacto crítico

Fuga de API keys de integraciones

Probabilidad baja. Mitigación: encriptación at rest + rotación + audit trail.
